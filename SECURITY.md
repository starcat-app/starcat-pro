# Security Policy

## Reporting a vulnerability

Report compromised download links, release-note tampering, impersonation, phishing, or unsafe public assets through [GitHub Security Advisories](https://github.com/starcat-app/starcat-pro/security/advisories/new). Do not include credentials, license keys, customer information, or exploit details in public issues.

The current default branch and the links published on `https://starcat.ink` are supported. Application or backend vulnerabilities should be reported against the repository that owns the affected code when it is known.
